# 信源查证与脚注规范

## 信源优先级

从高到低排列：

| 等级       | 信源类型               | 示例                                 |
| ---------- | ---------------------- | ------------------------------------ |
| **第一级** | 政府官方数据、维基百科 | 日本内阁府、总务省、中国统计局       |
| **第二级** | 官方机构、知名媒体     | JNTO、行业协会、第一财经、彭博、路透 |
| **第三级** | 学术论文、专业行业报告 | Google Scholar、知网、麦肯锡报告     |
| **避免**   | 无来源博客、不知名网站 | 个人公众号、营销软文                 |

## 查证策略

1. **先查权威信源** - 从第一级开始找
2. **信源不够则继续** - 找到更权威的来源
3. **提取关键数据** - 记录数据和计算逻辑
4. **质疑时升级** - 用户质疑信源可靠性时，立即寻找更高等级来源

## 数据使用原则

### 1. 不能只扔数据

必须有解释和语境，与个人体验结合。

| 写法                                                      | 评价     |
| --------------------------------------------------------- | -------- |
| ❌「日本智能马桶普及率高达 80%，TOTO 卖出超 6000 万台。」 | 生硬堆砌 |
| ✅「听说日本八成家庭都有智能马桶，这下算是体验到了。」    | 自然融入 |

### 2. 体感判断需要支撑

如「每三五个人就有一个中国人」这类判断，脚注中说明推算逻辑：

- 人口基数
- 游客数据
- 场景特点

### 3. 科普内容要精简

| 写法                                                                                          | 评价                 |
| --------------------------------------------------------------------------------------------- | -------------------- |
| ❌「日本《建筑基准法》规定特定建筑必须每 3 年进行一次外墙小型检查，每 10 年进行一次全面检查」 | 像教科书             |
| ✅「日本有规定建筑必须定期检查和维护外墙」                                                    | 精简，详细信息放脚注 |

### 4. 历史时间线要概括

| 写法                                     | 评价     |
| ---------------------------------------- | -------- |
| ❌「1964 年 XX，1967 年 XX，1972 年 XX」 | 堆砌年份 |
| ✅「六七十年代就开始建设了」             | 概括     |

## 脚注格式

### 正文写法

```markdown
听说日本八成家庭都有智能马桶[^1]，这下算是体验到了。
```

### 脚注写法

```markdown
[^1]: [数据来源](链接) - 根据日本内阁府调查，2014年普及率为76%，目前已超过80%。
```

### 脚注内容

- 权威信源链接
- 关键数据摘要
- 必要的计算逻辑（如体感判断的推算过程）

## 工作节奏

查一个 → 讨论 → 写一个 → 确认 → 下一个

对于每个需要查证的点（`<!-- TODO: 补充XXX -->`）：

1. **查找资料** - 按信源优先级查找
2. **与用户讨论写法** - 提出建议，用口语化表达
3. **根据反馈迭代** - 「太生硬」重新组织，「信源不够」重新查找
4. **写入文章** - 正文精简，详细信息放脚注
5. **等待确认** - 用户满意后才处理下一个
