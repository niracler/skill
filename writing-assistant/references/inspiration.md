# 启发模式

帮助用户在不知道写什么或如何写下去时，通过启发式提问引导写作。

## 核心原则

1. **不要一股脑问完** - 设计启发问题的 TODOList，逐个提问
2. **确认再继续** - 用户想好 → 写好 → 改好 → 确认后，才到下一个问题
3. **启发而非代写** - 用提问启发思考，不替用户决定内容方向

## 日记 / 周记 / 月记

### 日记框架

```text
1. Journal（日常记录）
   - Daily：今天做了什么？
   - Photo：有什么值得记录的画面？
   - Voice：有什么想说的话？
   - TIL：今天学到了什么？

2. Work Log（工作日志）
   - New Initiatives：启动了什么新事项？
   - Challenges Encountered：遇到了什么挑战？
   - Tasks Completed：完成了什么任务？

3. Today I Learn
   - 今天看了什么作品（漫画、动画、文章）？
   - 印象、亮点、反思？
```

### 日记启发问题

#### Daily

- 今天最让你印象深刻的一件事是什么？
- 有没有什么意外发生？
- 今天的心情如何？为什么？

#### Work Log

- 今天在工作上有什么进展？
- 有没有遇到什么卡住的地方？
- 有没有学到什么新东西？

#### TIL

- 今天看/读/听了什么？
- 有什么让你「原来如此」的地方？
- 有什么想吐槽或称赞的？

### 周记 / 月记扩展

在日记框架基础上增加：

- **Review**：这周/月的整体状态如何？
- **Highlights**：最值得记录的 3 件事？
- **Lowlights**：最想吐槽的 3 件事？
- **Next**：下周/月想做什么？

## 游记

### 游记框架

```text
1. 出发前
   - 为什么去这个地方？
   - 有什么期待？

2. 旅途中
   - 按时间线或地点组织
   - 感官体验（看到、听到、闻到、尝到）
   - 文化冲击 / 意外发现
   - 心理变化

3. 回来后
   - 最深刻的印象是什么？
   - 有什么改变了你的认知？
   - 会推荐别人去吗？为什么？
```

### 游记启发问题

#### 感官

- 到达时第一眼看到什么？
- 有什么气味让你印象深刻？
- 吃到了什么特别的东西？

#### 体验

- 有什么让你惊讶的地方？
- 有什么和你预期不同的？
- 有什么尴尬或有趣的经历？

#### 反思

- 这次旅行让你对什么有了新认识？
- 如果重来一次，你会怎么安排？
- 有什么遗憾或惊喜？

## TIL（Today I Learned）技术记录

### TIL 框架

```text
1. 背景
   - 遇到了什么问题？
   - 为什么需要解决？

2. 过程
   - 尝试了什么方法？
   - 踩了什么坑？

3. 解决方案
   - 最终怎么解决的？
   - 关键代码/配置是什么？

4. 收获
   - 学到了什么？
   - 下次遇到类似问题怎么办？
```

### TIL 启发问题

- 这个问题是怎么发现的？
- 你最开始以为是什么原因？
- 搜索了什么关键词？
- 哪个回答/文档帮到了你？
- 有没有什么坑需要提醒未来的自己？

## 通用文章

### 通用框架

```text
1. 起因
   - 为什么想写这个话题？
   - 有什么触发点？

2. 观点
   - 你的核心观点是什么？
   - 有什么支撑这个观点的经历/数据？

3. 展开
   - 有哪些角度可以讨论？
   - 有没有反面观点？

4. 收尾
   - 你希望读者带走什么？
   - 有什么行动建议？
```

### 通用启发问题

#### 破题

- 如果用一句话概括这篇文章，你会怎么说？
- 你最想让读者知道的一点是什么？

#### 展开

- 有没有具体的例子可以说明？
- 有没有人会反对你的观点？他们会怎么说？
- 你是怎么得出这个结论的？

#### 收尾

- 写完这篇文章，你自己有什么新认识？
- 有没有什么还没想清楚的地方？

## 流程示例

```text
1. 识别文章类型（日记/游记/TIL/通用）
2. 列出启发问题 TODOList
3. 逐个提问：
   Claude: 「今天最让你印象深刻的一件事是什么？」
   用户: [写下回答]
   Claude: [确认/追问] → 「好的，下一个问题：...」
4. 全部回答完后，整理成文
5. 进入审校模式打磨
```
