# 信源查证与脚注规范

## 信源优先级

从高到低排列：

| 等级   | 标记 | 信源类型           | 示例                               |
| ------ | ---- | ------------------ | ---------------------------------- |
| **L1** | 🔬   | 原始来源、官方公告 | 政府统计数据、官方文档、维基百科   |
| **L2** | 📰   | 权威媒体、研究机构 | 第一财经、路透、学术论文、行业报告 |
| **L3** | 📝   | 行业媒体、知名作者 | 行业协会、知名博主、专业公众号     |
| ⚠️     | 💬   | 尽量避免           | 个人博客、社交媒体讨论、营销软文   |

## 查证策略（SIFT）

遇到需要引用的信息时，按以下步骤验证：

| 步骤            | 行动     | 说明                                       |
| --------------- | -------- | ------------------------------------------ |
| **S**top        | 暂停     | 先停下来，不要直接复制引用                 |
| **I**nvestigate | 查背景   | 这个来源是谁？有什么立场或利益关系？       |
| **F**ind better | 找更好的 | 有没有更权威的来源（L1/L2）说同样的事？    |
| **T**race       | 追溯原始 | 这个说法的原始出处是什么？是否有断章取义？ |

**补充规则**：

- 从 L1 开始找，找不到再降级
- 用户质疑信源可靠性时，立即寻找更高等级来源
- 提取关键数据时，记录计算逻辑

## 数据使用原则

### 1. 不能只扔数据

必须有解释和语境，与个人体验结合。

| 写法                                                      | 评价     |
| --------------------------------------------------------- | -------- |
| ❌「日本智能马桶普及率高达 80%，TOTO 卖出超 6000 万台。」 | 生硬堆砌 |
| ✅「听说日本八成家庭都有智能马桶，这下算是体验到了。」    | 自然融入 |

### 2. 体感判断需要支撑

如「每三五个人就有一个中国人」这类判断，脚注中说明推算逻辑：

- 人口基数
- 游客数据
- 场景特点

### 3. 科普内容要精简

| 写法                                                                                          | 评价                 |
| --------------------------------------------------------------------------------------------- | -------------------- |
| ❌「日本《建筑基准法》规定特定建筑必须每 3 年进行一次外墙小型检查，每 10 年进行一次全面检查」 | 像教科书             |
| ✅「日本有规定建筑必须定期检查和维护外墙」                                                    | 精简，详细信息放脚注 |

### 4. 历史时间线要概括

| 写法                                     | 评价     |
| ---------------------------------------- | -------- |
| ❌「1964 年 XX，1967 年 XX，1972 年 XX」 | 堆砌年份 |
| ✅「六七十年代就开始建设了」             | 概括     |

## 脚注格式

### 正文写法

```markdown
听说日本八成家庭都有智能马桶[^1]，这下算是体验到了。
```

### 脚注写法

格式：`等级标记 | 链接 - 摘要`

```markdown
[^1]: 🔬 L1 | [日本内阁府](链接) - 2014年普及率为76%，目前已超过80%。

[^2]: 📰 L2 | [第一财经报道](链接) - 相关市场分析数据。
```

### 脚注内容

- 等级标记（🔬 L1 / 📰 L2 / 📝 L3）
- 权威信源链接
- 关键数据摘要
- 必要的计算逻辑（如体感判断的推算过程）

## 工作节奏

查一个 → 讨论 → 写一个 → 确认 → 下一个

对于每个需要查证的点（`<!-- TODO: 补充XXX -->`）：

1. **查找资料** - 按信源优先级查找
2. **与用户讨论写法** - 提出建议，用口语化表达
3. **根据反馈迭代** - 「太生硬」重新组织，「信源不够」重新查找
4. **写入文章** - 正文精简，详细信息放脚注
5. **等待确认** - 用户满意后才处理下一个
