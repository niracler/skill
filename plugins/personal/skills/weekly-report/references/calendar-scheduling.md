# Apple Calendar 排期

只在周报写回 Obsidian 后读取本文件。

## 工作日与容量

- 排期周固定为「本次报告目标周的下一周」，不是执行时所在自然周的下一周。
- 排期周类型与本次报告目标周相反。
- 排期周为大周时使用周一至周六，最多 6 个时间块。
- 排期周为小周时使用周一至周五，最多 5 个时间块。
- 每个工作日最多一个时间块，固定为 14:00～17:00。
- 一个计划日对应一个 3 小时时间块，不代表全天工时。
- 不额外查询法定节假日或调休；已有休假或节假日事件按冲突处理。

## 任务分配

按周报中的项目和任务顺序分配连续工作日：

- 正整数 `N` 对应连续 `N` 个工作日下午。
- 一个项目只有一条任务时，将项目的 `N 天` 全部分配给该任务。
- 一个项目有多条任务但没有逐条天数时，列为「需要拆分天数」并请求分配，不自行均分。
- 时间估算缺失或不是正整数时，列为「未排期」。
- 剩余工作日少于 `N` 时，先为可用日期生成时间块，再把未分配的天数列为「未排期」。
- 超出当周容量的部分保留在周报中，不创建日程。

## 日历与冲突

先读取实际日历名称：

```bash
osascript -e 'tell application "Calendar" to get name of calendars'
```

使用者未指定目标日历时，若列表中存在精确名称 `Work`，默认选用该日历；否则请求选择。
不要匹配带前后空格的名称，也不要使用近似匹配。使用者明确指定的日历优先于默认值。

先用本次报告目标周的周一加 7 天，得到排期周周一的绝对日期。即使补写历史周报，也必须
沿用这个日期，不得从执行当天重新判断「下周」。AppleScript 以 `current date` 的本地零点
为基准，加上排期日期与当天之间的整数天数差，构造候选日期，再设置 14:00 和 17:00。

对每个候选时间块查询目标日历中满足 `existing start < proposed end` 且
`existing end > proposed start` 的事件。存在冲突时：

1. 使用匿名任务名展示冲突日期和时间，不回显无关事件的敏感内容。
2. 暂停创建并等待使用者调整任务顺序、目标日期或保留为未排期。
3. 不覆盖或移动已有日程，也不自动向后顺延。

## 确认与创建

创建前展示完整建议：

```text
下周日程建议（目标日历，14:00-17:00）：
  周一：项目 A - 任务 1
  周二：项目 A - 任务 1
  周三：项目 B - 任务 2

冲突：
  周四 14:00-17:00：已有日程，不创建

未排期：
  项目 C - 需要拆分天数
```

不存在未处理冲突，或使用者明确决定把冲突任务保留为未排期后，再展示最终方案。取得明确
确认后，只创建无冲突且已分配的时间块。事件标题统一为
`{项目}：{加粗任务名称}`。建议中必须显示排期周的完整日期范围；创建后重新查询并核对
标题、开始时间和结束时间。
